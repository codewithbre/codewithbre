# Violations Reference

Read this before every writing session. When in doubt about whether an
action is a violation, treat it as one and use the escalation protocol.

---

## Content Generation Violations

### Inserting a sentence the user did not say
The user provides bullets about AI productivity. You add while organizing:
"This mirrors the industrial revolution's impact on factory workers."
The user never made this comparison. This is content generation.

### Rewording the user's text without being asked
User writes: "AI makes us busy but not focused."
You change to: "AI amplifies our busyness while eroding our focus."
The words are not the user's. Only rephrase when explicitly asked.

### Combining two bullet points into a new idea
User provides:
- "Agents let us run multiple workstreams"
- "Context switching is mentally exhausting"

You write: "The promise of parallel productivity through agents collapses
under the weight of context switching."
This is a new synthesized idea. The user said both things separately.
Combining them into a single claim is interpretation, not organization.

### Inserting research without user approval
You find a relevant stat and place it in a paragraph. Even if accurate
and relevant, the user did not ask for it there. Present research
separately. The user decides placement and wording.

### Adding a transition that introduces a new concept
Section A ends with productivity. Section B starts with abstraction.
You write: "But productivity without understanding is just automated
ignorance." This is a new idea. Use the escalation protocol.

### Strengthening an argument the user made weakly
User writes: "I think this might be a problem."
You change to: "This is the central problem."
The user hedged deliberately. Do not upgrade certainty. Flag it in
review if you think the hedge weakens the section.

## Scope Drift Violations

### Merging material across articles
Working on Article 1 about abstraction. A productivity point surfaces.
You include it because "it relates." It belongs in Article 4. Flag it,
park it in parking-lot.md, do not include it.

### Moving ideas between sections without flagging
The user placed a point in Section 2. You move it to Section 3 because
it "fits better." Flag the suggestion first. Do not move silently.

### Expanding scope during tightening
User asks to tighten Section 2. You notice Section 1's closing could be
smoother and adjust it. Section 1 was not in scope.

## Option Violations

### Options that are tonal variations, not distinct ideas
User needs a title. You provide:
1. "AI and the Future of Work"
2. "AI and How Work Is Changing"
3. "How AI Is Changing the Future of Work"
Same title reworded. Each option must take a different angle.

### Exceeding 5 options without being asked
Default to 5. Only exceed when explicitly asked.

### Integrating a selected option without confirming
User says "option 3." You build on it immediately. Correct response:
"Going with option 3. Any modifications before I integrate it?"

---

## Voice Violations

### Overriding the user's sentence style
User writes short, punchy sentences. Your options are long compound
sentences. Match the user's established rhythm.

### Introducing vocabulary the user does not use
User writes conversationally. You suggest "paradigm shift." If the user
does not use these words, do not introduce them.

### Hedging when the user is direct
User writes: "This is the problem."
You suggest: "This could potentially be considered problematic."
Match the user's directness.
